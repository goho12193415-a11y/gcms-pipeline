每个样品输出一个独立的 Excel 文件（文件名 = 样品名）。一次处理多个样品时
会生成多个 Excel，互不合并。每个文件含三个工作表，按此顺序查看最省力：
  「汇总」→「待复核」→「Results」

支持 Thermo .RAW / .mzML 与岛津 GCMSsolution .qgd 数据。


一、汇总 工作表（先看这张，掌握全局）
--------------------------------------
一眼看清各类峰的数量：

  峰总数
  可信免审（绿+RI双证据确认）   ← 这些不用人工看
  ★待复核                      ← 实际需要核对的，去"待复核"表
  微量峰（<0.05%，免复核）      ← 面积太小，通常不报
  污染物（红，可跳过）
  低置信（灰，信号弱，可略看）
  — RI核对 OK / 可疑 / 无文献RI 的数量

各类相加 = 峰总数。审核工作主要就盯"待复核"那一栏的数字。


二、待复核 工作表（日常审核主要看这张）
----------------------------------------
只列需要人工核对的峰，已按面积从大到小排序（大峰优先看）：
  - 黄色（REVIEW）里"没有 RI 佐证"的峰
  - 任何"RI 可疑"的峰（质谱说是、RI 说不是——正是要抓的）

以下峰不在此表（已自动归类，无需逐个翻）：
  - 绿色自动确认、RI 双证据确认（OK）的 → 可信
  - 微量峰（面积 < 0.05%）→ 噪声/痕量，通常不报
  - 红色污染物、灰色低置信

列：Peak_No、RT_min、Area、Percentage(面积%)、Status、RI_Check、
    Compound_Name、RMF、FMF、RI_WAX、CAS

→ 从"翻几百个峰"变成"看这张表里的二十来个关键峰"。


三、Results 工作表（全部峰的全部候选，需要细查时看）
----------------------------------------------------
每个色谱峰最多 8 行（Rank 1-8，对应前八个候选）。列名及含义：

  Sample           样品名称
  Peak_No          峰序号
  RT_min           保留时间（分钟），峰顶位置
  RT_start/RT_end  峰起始/结束时间（分钟）
  Area / Height    峰面积 / 峰高
  SN               信噪比
  Percentage       相对含量（面积归一化百分比）
  Status           审核状态（见第四节）
  RI_Check         实测 RI 与文献的核对结果（见第五节）
  User_Confirm     人工确认列（空白，供用户填写）
  Rank             候选排名（1=最佳匹配 … 8）
  Compound_Name    候选化合物名称
  CAS              CAS 号
  FMF              正向匹配因子（0-999）。样品谱图与库谱的匹配程度
  RMF              反向匹配因子（0-999）。排除干扰离子后的匹配程度
  RI_WAX / RI_DB5  该化合物在 DB-WAX / DB-5 柱上的文献保留指数
  RI_Diff          文献 RI 与"本柱校准线预测值"的残差（仅当文献值存在时）。
                   与 RI_Check 用同一套标定，二者一致：|RI_Diff|≤30 → OK，
                   偏大 → suspect。不是实测与文献的原始差（那会因柱型不同而
                   系统性偏大，看似与 RI_Check 矛盾）。
  Source           候选来源（replib = 来自 NIST 复本库的补充候选，常在 6-8 名）
  Coelution        标记是否疑似共流出（Y=是）
  Food_In_Top5     标记该峰前 5 候选中是否含常见食品挥发物


四、审核状态（Status）颜色含义
------------------------------
  绿色（CONFIRMED）：自动确认（RMF>900 且 FMF>700，RI 吻合或为已知食品挥发物）
                     → 可跳过或快速扫一眼
  黄色（REVIEW）：    待人工审核 → 重点看（但有 RI 佐证 OK 的已免审，见下）
  红色（CONTAMINANT）：疑似污染物（硅氧烷/塑化剂/BHT 等）→ 通常可跳过
  灰色（LOW）：       低置信度（RMF<750 或 FMF<400）→ 大部分可跳过


五、RI_Check 列含义（需在界面提供"RI 标品"才有意义）
----------------------------------------------------
软件用本样品自身的高置信命中，自标定出"本柱 RI ↔ 文献 RI"的关系，
再核对每个 top-1 鉴定：

  OK            质谱 + 文献 RI 双证据一致 → 可信，无需人工再审
  suspect       RI 与该鉴定矛盾 → 务必复核（哪怕质谱分高）
  OK-pred       无文献 RI，改用 CNN 预测 RI 且一致 → 弱证据，仅参考
  suspect-pred  无文献 RI，预测 RI 矛盾 → 弱证据
  noRI          既无文献也无预测 RI，无法核对
  noCal         本样品可校准的化合物太少，未做 RI 标定（没选标品时也是此值）

★ 标品须与样品"同批次/同期"测定：保留时间会随柱子老化漂移，
  标品与样品相隔太久（如数月），RI 标定不准，RI_Check 可能误判。


六、如何审核一个"待复核"峰
--------------------------
  1. 看 RI_Check：若该峰其它候选有 OK，优先采信；若 suspect，重点查。
  2. 看 RMF：>850 质谱较可靠；750-850 需结合 RI；<750 谨慎。
  3. 看 RI_Diff：<30 吻合好；30-80 尚可；>80 或无值 谨慎。
  4. 交叉验证：RI_WAX 与 RI_DB5 两边都吻合更可靠。
  5. 化学合理性：该化合物是否可能出现在本样品中？
  6. 在 User_Confirm 列填标记（如 √），保存。


七、根据 RMF 判断可靠性（经验参考）
-----------------------------------
  950-999  极高（几乎确定）   900-949  很高（需 RI 确认）
  850-899  高（需 RI+经验）   800-849  中等（RI 必须吻合）
  750-799  一般（需多证据）   700-749  偏低（仅作参考）


八、常见问题
-----------
Q: 为什么同一个峰有多行？
A: 软件为每个峰输出前 8 个候选（Rank 1-8）供比对；第 6-8 名可能含复本库
   补充候选（Source=replib）。

Q: RI_Check 全是 noCal？
A: 未选 RI 标品，或本样品可标定的化合物太少。选上同期标品即可。

Q: RMF 很高但 RI_Check=suspect？
A: 质谱匹配好不等于鉴定正确，可能是相近化合物。RI 不吻合时优先存疑、复核。

Q: RI_WAX 列为空？
A: 该候选在 NIST 文献中缺 DB-WAX 保留指数。可参考 RI_DB5 或经验判断。

Q: 候选都不合理？
A: 峰面积大时可在 NIST MS Search 中重搜该峰，或在 User_Confirm 标 "Unknown"。
